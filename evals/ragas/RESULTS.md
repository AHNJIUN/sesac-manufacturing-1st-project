# RAG 평가 결과 리포트 (Evidence Agent)

생성된 평가셋(`dataset.jsonl`, 29문항)으로 **검색(IR) + 생성품질(RAGAS) + 예측정확도**를 측정한 결과와 해석.

- 평가 대상: `evidence_agent`의 RAG 경로(`rag_service.rag_search`) + 1-2는 `prediction_service.run_prediction` → mode B
- 벡터 백엔드: Pinecone `sesacline-agent-docs` (Haas 문서 색인)
- 임베딩: `text-embedding-3-small` / RAGAS judge·GT생성: `gpt-4o-mini`
- **`MIN_EVIDENCE_SCORE=0.3`** (기본 0.45에서 낮춰 커버리지↑)
- 재현: `build_samples.py` → `run_ragas.py`(RAGAS) / `ir_eval.py`(IR)

## 5. 해석 (핵심 발견)

1. **검색이 발동하면 근거 충실도는 높다** — faithfulness 0.92, context_precision 0.81. 가져온 근거에 답이 잘 정합(환각 적음).
2. **1-2(예측 경로)가 가장 강함** — context_precision **1.00**, recall 0.66, faithfulness 0.99, **예측 정확도 8/8**. taxonomy mode B 라우팅이 정확한 문서로 보냄.
3. **answer_relevancy ~0.13, 0.00 다수 = 포맷 아티팩트** — `EVIDENCE_SUMMARY_SYSTEM`의 citation 불릿 + "담당자 확인 필요" 헤징을 RAGAS가 **noncommittal(회피)로 판정해 0점** 처리. faithfulness가 높은 점이 "관련성이 진짜 낮은 게 아님"을 방증.
4. **IR 절대값이 RAGAS보다 낮아 보이는 건 청킹 불일치 탓** — gold(`reference_contexts`, 1000자)와 색인 청크(1200자)가 **다른 경계**라 진짜 출처 매칭도 코사인이 눌림(0.66대). 따라서 **IR @K는 절대 recall이 아니라 하한(lower bound)**으로 읽어야 함. RAGAS(LLM judge)는 텍스트 의미로 보므로 더 관대.
5. **1-2 IR은 참고용(비신뢰)** — 한-영 교차언어 + 짧은 GT라 신호(0.42~0.47)와 무관(0.38) 분리도가 낮음. 1-2 검색 품질은 **RAGAS context_recall(0.66)·precision(1.00)** 과 **예측 정확도(8/8)** 로 판단하는 게 타당.
6. **저(低)품질 개별 케이스**: faithfulness 낮음 `k11_10(0.52)·k11_14(0.58)·k11_21(0.60)` — 근거 약한 문장 혼입 점검 대상. 검색 0점 `k11_01·05·06·07·09·17·18·21` — 질문이 코퍼스 핵심에서 벗어났거나 서술형 유사질문.

## 7. 한계 & 다음 단계

- **GT 사람 검수**: 1-1·1-2 GT 모두 LLM 생성 초안 → 검수해야 RAGAS·IR 신뢰도↑.
- **청킹 정렬**: IR을 절대값으로 쓰려면 gold를 **색인과 동일한 청킹**으로 라벨링하거나, 문서/섹션 단위 relevance로 평가.
- **answer_relevancy 검증**: answer를 evidence 요약 대신 **final_answer**로 바꿔 재측정하면 0.00이 포맷 탓인지 확정.
- **1-2 IR 대체지표**: 교차언어 임베딩 대신 LLM-judge 관련성 라벨로 Precision@K·MRR 산출 검토.
- **카테고리 균형**: 1-1이 maintenance 편중 → cause/inspection 질문 보강(페르소나/문서 확대) 고려.
